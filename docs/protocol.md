# Протокол репутации repOWR

## Описание

Протокол `repOWR` (Reputation Open World Republic) — децентрализованная система репутации на базе блокчейна TON. Оценки и данные профилей хранятся в комментариях Jetton-транзакций, что обеспечивает прозрачность и неизменяемость репутационной истории.

---

## Форматы сообщений

Протокол поддерживает **два формата** для удобства пользователей:

### 1. Упрощённый формат (для быстрых отзывов)

**Синтаксис:**
```
repOWR:RATING:COMMENT:
```

**Правила:**
- `repOWR` — идентификатор протокола (обязательно)
- `RATING` — оценка от 1 до 5 (обязательно)
- `COMMENT` — текст отзыва (опционально)
- **Закрывающее двоеточие обязательно** — оно обозначает конец сообщения
- Комментарий берётся между вторым и последним двоеточием, поэтому двоеточия внутри комментария использовать **нельзя**

**Примеры:**
```
repOWR:5:
repOWR:5:Отлично!:
repOWR:4:Хорошая работа, быстро и качественно:
repOWR:3:Неплохо, но есть недочёты:
```

**Преимущества:**
- Минимум символов для ввода
- Быстро набирается в мобильном кошельке
- Не требует знания JSON синтаксиса

---

### 2. JSON-формат (для расширенных данных)

**Базовая структура оценки:**
```json
{
  "protocol": "repOWR",
  "rating": 5
}
```

**Полная структура оценки:**
```json
{
  "protocol": "repOWR",
  "rating": 5,
  "type": "deal",
  "ref": "0:abc123...",
  "comment": "Отличная работа!",
  "link": "https://proof.com"
}
```

**Поля оценки:**

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `protocol` | string | ✅ Да | Всегда `"repOWR"` |
| `rating` | integer | ✅ Да | Оценка от 1 до 5 |
| `type` | string | ❌ Нет | Тип взаимодействия: `"deal"`, `"service"`, `"product"`, `"general"` |
| `ref` | string | ❌ Нет | Адрес TON кошелька третьей стороны (арбитр, платформа) |
| `comment` | string | ❌ Нет | Текстовый отзыв (до 500 символов) |
| `link` | string | ❌ Нет | Ссылка на подтверждение (до 200 символов) |

## Профиль пользователя (Identity)

JSON-формат для создания децентрализованного профиля/паспорта пользователя.

### Упрощённый формат (для кошелька Telegram)

Кошелёк Telegram ограничивает комментарий 54 символами. Для таких случаев поддерживается минимальный формат:

```
repOWR:identity:ВашНик:
```

**Правила:** `identity` — ключевое слово; никнейм — максимум 37 символов; закрывающее двоеточие обязательно.

**Пример:** `repOWR:identity:Александр:` (26 символов)

## Типы транзакций

### 1. Отправка оценки (Rating)
- **Отправитель:** тот, кто оценивает
- **Получатель:** тот, кого оценивают
- **Сумма:** любая (минимальной суммы нет — можно отправить 0.001 SPW с критикой или 100 SPW с благодарностью)
- **Комментарий:** упрощённый или JSON формат

### 2. Создание профиля (Identity)
- **Отправитель:** владелец профиля
- **Получатель:** адрес ОМР `UQCywBj5RIyKYf1SeLMkmt9gL13pMzCaqORZZ3iFeJyoRaqO`
- **Сумма:** любая (лимиты на обновление профиля могут быть введены в будущем)
- **Комментарий:** JSON с полями профиля

### Обновление профиля

Профиль можно изменить в любой момент, отправив новую identity-транзакцию с того же кошелька. Бот и виджеты всегда отображают последнюю версию профиля. Предыдущие версии остаются в блокчейне, но не используются системой.

## Валидация

### Упрощённый формат
```python
def validate_simple_format(comment: str) -> bool:
    """Проверка упрощённого формата"""
    # Должен начинаться с "repOWR:"
    if not comment.startswith("repOWR:"):
        return False
    
    # Должен заканчиваться двоеточием
    if not comment.endswith(":"):
        return False
    
    # Извлекаем части
    parts = comment[7:-1].split(":", 1)  # Убираем "repOWR:" и последнее ":"
    
    # Проверяем рейтинг
    try:
        rating = int(parts[0])
        if not 1 <= rating <= 5:
            return False
    except (ValueError, IndexError):
        return False
    
    return True
```

### JSON-формат
```python
def validate_json_format(comment: str) -> bool:
    """Проверка JSON формата"""
    try:
        data = json.loads(comment)
        
        # Обязательные поля
        if data.get("protocol") != "repOWR":
            return False
        
        # Для оценок
        if "rating" in data:
            rating = data["rating"]
            if not isinstance(rating, int) or not 1 <= rating <= 5:
                return False
        
        # Для профилей
        if data.get("type") == "identity":
            if not data.get("nickname"):
                return False
        
        return True
        
    except json.JSONDecodeError:
        return False
```

## Расчёт репутации

Система расчёта репутации анализирует все полученные оценки и формирует итоговый балл пользователя.

### Базовые метрики репутации
- **Средний рейтинг:** среднее арифметическое всех оценок (от 1 до 5)
- **Количество оценок:** общее число полученных отзывов
- **Отзывов оставлено:** количество оценок, выставленных пользователем

### Social Power Score

Помимо среднего рейтинга, каждый пользователь имеет **Social Power Score** (SP) — комплексный показатель влияния в системе. Максимальное значение: **2000 SP**.

Social Power складывается из пяти компонентов:

| Компонент | Формула | Максимум |
|-----------|---------|---------|
| SPW баланс | √SPW × 15 + log10(SPW) × 50 | 800 |
| Оставленные отзывы | ступенчатая шкала | 500 |
| Полученные отзывы | ступенчатая шкала | 400 |
| Заполненность профиля | по 10–20 очков за каждое поле | 100 |
| Возраст аккаунта | 0–100 в зависимости от дней с первой транзакции | 100 |

Первые 100 зарегистрированных пользователей получают дополнительный бонус **+30 SP**.

### Ранги по Social Power

| Минимальный SP | Ранг | Вес голоса |
|----------------|------|-----------|
| 1500 | 🏆 Легенда | 10x |
| 1000 | 👑 Влиятельный | 5x |
| 600 | 💎 Эксперт | 3x |
| 300 | ⚡ Активист | 2x |
| 100 | 📍 Участник | 1.5x |
| 0 | 🌱 Новичок | 1x |

**Примечание:** Алгоритм расчёта может совершенствоваться для выявления паттернов накрутки и улучшения качества оценки репутации.

## Защита от мошенничества

### 1. Минимальная стоимость оценки
- Каждая оценка требует реальной транзакции с минимальной суммой
- Это делает спам экономически невыгодным

### 2. Временные метки
- Все транзакции имеют timestamp в блокчейне
- Можно отслеживать подозрительные паттерны активности

### 3. Анализ связей
- Система анализирует граф взаимодействий между кошельками
- Выявляются подозрительные паттерны (взаимные оценки, массовые 5-звёздочные отзывы от новых кошельков)
- Алгоритмы обнаружения постоянно совершенствуются

### 4. Политика накрутки

Искусственная накрутка отзывов запрещена. Оценки, отправленные самому себе, считаются накруткой и не учитываются при расчёте репутации. Команда repOWR оставляет за собой право выявлять случаи манипуляции репутацией и отображать скорректированную оценку с учётом исключённых подозрительных транзакций. Исходные данные при этом остаются в блокчейне без изменений — корректировка происходит только на уровне отображения.

## Безопасность для пользователей

### Неизменяемость данных

Все данные, записанные в блокчейн (оценки, комментарии, профили), невозможно удалить или отредактировать. Перед отправкой транзакции убедитесь, что вы не раскрываете информацию, о которой можете пожалеть в будущем.

### Рекомендации по кошельку

Для работы с repOWR рекомендуется создать **отдельный кошелёк**, не связанный с основными средствами. Репутационный адрес будет публичным — его баланс и история транзакций доступны любому. Использование отдельного кошелька снижает риски в случае социальной инженерии.

### Персональные данные

**Не рекомендуется** указывать в профиле:
- Полное ФИО
- Номер телефона
- Точный адрес проживания
- Email основной почты
- Любые данные, позволяющие установить личность

Для связи в профиле достаточно указать псевдоним в Telegram или ник на GitHub. Протокол не запрещает размещение любых данных, но ответственность за последствия несёт пользователь.

### Предупреждение

Комбинация публичного баланса кошелька и персональных данных в профиле может сделать владельца мишенью для мошенников. Будьте осмотрительны.

## Токеномика SPW

Протокол repOWR работает на базе токена **SPW** (Jetton на блокчейне TON).

### Эмиссия

Общий выпуск: **100,000,000 SPW** (фиксированный, дополнительная эмиссия не предусмотрена).

### Распределение

| Категория | Доля | Количество SPW | Условия |
|-----------|------|---------------|---------|
| Команда и развитие | 15% | 15,000,000 | Lock 2 года: cliff 6 месяцев, затем линейная разблокировка помесячно |
| Ликвидность DEX | 7% | 7,000,000 | Без заморозки, пополняется по мере роста проекта |
| Фонд экосистемы | 35% | 35,000,000 | Lock 3 года: линейная разблокировка помесячно |
| Комьюнити и маркетинг | 13% | 13,000,000 | Lock 1.5 года: линейная разблокировка помесячно |
| Резерв | 20% | 20,000,000 | Lock 3 года: cliff 1 год, затем разблокировка поквартально |
| Сжигание | 10% | 10,000,000 | Сжигается по мере появления механизмов |

Каждая категория хранится на отдельном кошельке. Адреса всех кошельков публичны и доступны для проверки в блокчейне.

### Назначение фондов

**Команда и развитие** — токены создателя проекта. Строгий lock с cliff 6 месяцев демонстрирует долгосрочную приверженность проекту.

**Ликвидность DEX** — обеспечение торгового пула на STON.fi. Токены не замораживаются, чтобы сохранить гибкость при управлении ликвидностью.

**Фонд экосистемы** — награды за использование системы репутации, гранты разработчикам, интеграции сторонних сервисов, пополнение пула ликвидности.

**Комьюнити и маркетинг** — эирдропы для первых пользователей, конкурсы, партнёрства с другими проектами на TON.

**Резерв** — стратегический запас на непредвиденные нужды, будущие листинги, партнёрства.

**Сжигание** — дефляционный механизм для постепенного сокращения общего предложения токенов.

### Стратегия ликвидности

Пул ликвидности на STON.fi пополняется постепенно из фонда экосистемы по мере роста числа пользователей и объёма торгов. Это обеспечивает плавное увеличение глубины пула, снижает проскальзывание при сделках и не сдерживает органический рост цены токена. Рынок определяет цену SPW самостоятельно — задача проекта обеспечить достаточную ликвидность для комфортной торговли.

### Механизм сжигания (burn)

Дефляция привязана к активности в экосистеме. Планируемые триггеры сжигания:

- Платное обновление профиля (после первых бесплатных редакций) — 100% суммы на burn
- Комиссии за голосования — на burn
- Премиум-функции (верификация, расширенная аналитика) — частично на burn

Сжигание происходит путём отправки токенов на публичный burn-адрес. Объём сожжённых токенов доступен для проверки в блокчейне в любой момент.

### Стоимость транзакций

Минимальная сумма транзакции не установлена. Пользователи свободны отправлять любое количество SPW вместе с оценкой или профилем.

## Версионирование

- **Текущий протокол:** `repOWR`
- **Обратная совместимость:** протокол постоянно развивается с сохранением совместимости со всеми предыдущими форматами данных
- **Революционные изменения:** при необходимости принципиально нового протокола будет создан `repOWR2` или аналогичный

Философия протокола: эволюция без breaking changes. Парсер становится умнее, но старые данные всегда остаются читаемыми.
